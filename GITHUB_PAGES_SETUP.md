# GitHub Pages Deployment

## Access Yoda's Airhorn Online

Your application is now ready to be deployed to GitHub Pages!

### How to Enable GitHub Pages

1. Go to your repository on GitHub: https://github.com/nbonhamwwt/js-pairing-exercises
2. Click **Settings** (gear icon in the top right)
3. Scroll down to **Pages** section (on the left sidebar)
4. Under "Source", select:
   - **Branch**: `master`
   - **Folder**: `/ (root)`
5. Click **Save**

### Access Your Application

Once GitHub Pages is enabled, your airhorn will be live at:

🔗 **https://nbonhamwwt.github.io/js-pairing-exercises/**

(It may take 1-2 minutes for the first deployment)

### How It Works

- The root `index.html` file redirects to `public/index.html`
- GitHub Pages will serve your files directly from the repository
- Any changes pushed to master will automatically update the live site

### Direct Access

You can also access the app directly at:
- https://nbonhamwwt.github.io/js-pairing-exercises/public/

### Troubleshooting

**If the page doesn't load:**
1. Wait 1-2 minutes for GitHub to process
2. Hard refresh your browser (Ctrl+Shift+R or Cmd+Shift+R)
3. Check the Settings → Pages section to confirm it's enabled
4. Look at the Actions tab to see if there were any deployment errors

**If you see a 404 error:**
- Make sure you selected `/ (root)` as the source folder, not `/public`
